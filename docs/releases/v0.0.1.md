# NervusDB v0.0.1

This release is the first public release of the reset 0.1-core line.

It deliberately does **not** continue the old platform-era `v1.x` / `v2.x`
scope. Those GitHub releases are kept as legacy history. `v0.0.1` is the
current embedded Rust graph database baseline.

## What This Release Is

- One public crate: `nervusdb`.
- Fjall-backed local database directory storage.
- Rust embedded API for nodes, labels, relationship types, edges, properties,
  snapshots, and write transactions.
- Mini-Cypher 0.1 query surface.
- CLI smoke/debug/query/write workflows in the repository.
- Crash-safe persistence through Fjall.

Install:

```toml
[dependencies]
nervusdb = "0.0.1"
```

## Core Capabilities

- `Db::open(path)` opens a local database directory.
- Single-writer transactions.
- Lock-free read snapshots.
- Node and edge persistence.
- Label and relationship type interning.
- Label scans through the storage-level label index.
- Directed one-hop and documented two-hop traversal.
- Node and edge property read/write.
- Reopen durability.
- Basic `CREATE`, `SET`, `DELETE`, `MATCH`, `WHERE`, `RETURN`, `LIMIT`, and
  `EXPLAIN` in Mini-Cypher.

## Explicit Non-Goals

This release does not promise:

- old `.ndb + .wal` compatibility,
- full openCypher,
- property indexes,
- parallel edges,
- Python/Node/C bindings,
- vector/HNSW search,
- multi-writer OCC,
- production-scale benchmark guarantees.

Unsupported advanced query paths are intentionally rejected instead of kept as
half-working zombie features.

## Validation

Validated locally before release:

```bash

bash scripts/check.sh
cargo test -p nervusdb-storage --test core_0_1_storage
cargo test -p nervusdb --test core_0_1_rust_api
cargo test -p nervusdb --test core_0_1_examples
bash scripts/core_examples.sh
bash scripts/core_crash_recovery.sh
bash scripts/core_bench.sh --small
bash scripts/core_bench.sh --nodes 100000 --degree 5 --iters 1000
cargo test --workspace
cargo publish -p nervusdb --dry-run --registry crates-io
```

GitHub Actions for `main` passed before tagging.

bulk loading belongs in a later release.

## Packaging

Only `nervusdb` is published for 0.0.1. The workspace-local crates
`nervusdb-api`, `nervusdb-storage`, and `nervusdb-query` are `publish = false`
wrappers and are not separate public packages.
