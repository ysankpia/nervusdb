# NervusDB v0.0.3

This release keeps the single-crate 0.1-core product surface and tightens the
basic graph integrity rules.

It does not add property indexes, edge IDs, parallel edges, advanced Cypher,
vectors, multi-writer concurrency, or unsafe write modes.

## What Changed

- `create_edge(src, rel, dst)` rejects dangling edges when either endpoint is
  missing, tombstoned, or not created live in the same transaction.
- Node label and property mutations reject missing or tombstoned nodes.
- Edge property mutations and edge deletion reject missing edges.
- Direct Rust API `WriteTxn::tombstone_node(node)` now returns `Result<()>` and
  performs detach-clean deletion for the storage layer.
- Direct Rust API `WriteTxn::tombstone_edge(src, rel, dst)` now returns
  `Result<()>` and cleans edge properties.
- Node deletion cleans node properties, node labels, label scan keys, incident
  adjacency keys, and incident edge properties in the same commit batch.
- Repeated creation of the same `(src, rel, dst)` edge remains idempotent.
- Mini-Cypher keeps the visible distinction between `DELETE n` and
  `DETACH DELETE n`: plain `DELETE` still rejects connected nodes, while
  `DETACH DELETE` removes relationships and the node.

## Why It Matters

`v0.0.2` made bulk writes fast enough to expose the next real problem: graph
state must not drift into impossible shapes. A graph database that accepts
edges pointing to missing nodes, or leaves properties and adjacency keys behind
after deletion, is not a graph database; it is just loose key-value data.

`v0.0.3` makes the storage layer enforce the basic invariants before more
expensive features such as property indexes are designed. That ordering is
deliberate: indexes would amplify stale data if deletion and mutation rules were
not correct first.

## Install

```toml
[dependencies]
nervusdb = "0.0.3"
```

Only `nervusdb` is published. The workspace-local crates `nervusdb-api`,
`nervusdb-storage`, and `nervusdb-query` remain `publish = false` wrappers and
are not separate public packages.


## Compatibility Notes

This is a pre-0.1 line and intentionally allows API breakage. In this release,
direct Rust API deletion methods changed from infallible calls to `Result<()>`
because missing nodes and edges are real errors.

The database path remains a local directory. There is still no `.ndb + .wal`
public storage format, no old-format migration, no property index, and no
parallel-edge identity.

## Explicit Non-Goals

This release still does not include:

- property indexes,
- old `.ndb + .wal` compatibility,
- full openCypher,
- parallel edges or edge IDs,
- public bulk loader / durability options,
- unsafe buffered write mode,
- Python/Node/C bindings,
- vector/HNSW search.
