# NervusDB v0.0.8

This release is the performance closeout for the embedded graph core. It does
not add query features or change the public Rust API. It deliberately breaks
on-disk compatibility to move NervusDB to storage epoch 4.

The point of this release is narrow:

- repair the v0.0.7 traversal regression,
- keep clean reopen in the millisecond range,
- reduce adjacency write/read amplification without weakening durability,
- document the remaining durable commit floor honestly.

## Breaking Storage Change

`STORAGE_FORMAT_EPOCH` is now `4`.

NervusDB `v0.0.8` does not open `v0.0.7` database directories. Old directories
return `StorageFormatMismatch`. There is no migration tool in this release.
Users should rebuild, recreate, or export/reimport data.

## What Changed

- Added ADR 0010 for packed adjacency lists.
- Changed adjacency from one KV record per edge direction to one sorted packed
  list per `(node, rel)` pair:
  - `adj_out [src][rel] -> repeated dst:u32 BE`,
  - `adj_in [dst][rel] -> repeated src:u32 BE`.
- Kept the v0.0.7 four-keyspace physical shape:
  - `meta`,
  - `graph_data`,
  - `adj_out`,
  - `adj_in`.
- Kept public graph semantics unchanged:
  - edge identity remains `(src, rel, dst)`,
  - parallel edges remain unsupported,
  - duplicate edge creation remains idempotent,
  - Mini-Cypher behavior is unchanged.
- Updated storage commit to group adjacency writes by `(node, rel)` and write
  packed lists in the same durable Fjall batch.
- Updated traversal to use point reads for rel-qualified `neighbors` and
  `incoming_neighbors`.
- Updated fsck-lite to expand packed adjacency values when checking `adj_out` /
  `adj_in` symmetry.
- Changed explicit `Db::close()` to perform clean shutdown over the current
  Fjall backend: persist with `SyncAll`, flush keyspaces, release the handle,
  then checkpoint the active clean-shutdown journal. This avoids clean reopen
  replay without changing normal commit durability or crash recovery.
- Added storage coverage for epoch 4 keyspaces and close-time journal checkpoint.

## Interpretation

This release fixes the traversal regression from v0.0.7. The access grain now
matches the graph operation: common rel-qualified neighbor reads no longer pay a
small LSM prefix iterator setup for every five-edge adjacency list.

This release also keeps clean reopen cheap. The active Fjall journal can remain
below Fjall's internal rotation threshold after packed adjacency reduces write
bytes. Explicit `Db::close()` now handles that clean-shutdown case.

The remaining durable commit cost is not hidden. The profile shows the large
remaining stage is Fjall `SyncAll` batch persistence. NervusDB does not add an
unsafe buffered mode, public durability knob, or bulk loader in this release.

## Install

```toml
[dependencies]
nervusdb = "0.0.8"
```

Only `nervusdb` is published. The workspace-local crates `nervusdb-api`,
`nervusdb-storage`, and `nervusdb-query` remain `publish = false` wrappers and
are not separate public packages.

## Explicit Non-Goals

This release does not include:

- old database migration,
- single-file storage,
- unsafe buffered write mode,
- public bulk-loader or durability configuration,
- range indexes,
- public `Db::create_index` or `GraphSnapshot::lookup_index`,
- edge IDs or parallel edges,
- vector/HNSW search,
- multi-writer concurrency,
- full openCypher,
- Python/Node/C bindings,
- long-term storage-format compatibility promises before 0.1.
